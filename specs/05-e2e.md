# Spec 05: End-to-end integration

## Scope

The plumbing that makes all components work together as a system:
docker-compose, Makefile, configuration files, and an end-to-end test
that exercises the whole pipe.

## Deliverables

### `docker-compose.yml`

Brings up everything needed for local dev and e2e testing:

- `qdrant` — the vector store. Use the official `qdrant/qdrant` image.
- `ingest-gateway` — built from `packages/tilth-server/Dockerfile`.
  Entrypoint runs the ingest app.
- `query-gateway` — built from `packages/tilth-server/Dockerfile`.
  Entrypoint runs the query app.

Networking: gateways can reach Qdrant. Test runner (host) can reach the
gateways. The MCP server is started as needed by tests, not always-on.

Required env vars are set via `.env` (gitignored) or passed at runtime.
Provide `config/.env.example` with placeholder values.

### `Makefile`

Targets:

```
make install    # install all packages in editable mode + dev deps
make lint       # ruff check across all packages
make typecheck  # mypy --strict across all packages
make test       # pytest each package
make up         # docker-compose up -d (qdrant + gateways)
make down       # docker-compose down -v
make e2e        # bring stack up, run e2e tests, tear down
make clean      # rm caches, __pycache__, .pytest_cache, etc.
```

`make install` should work on a clean clone with only Python 3.11+ and
Docker installed. No manual steps.

In `config/`:

- `write-policy.yaml` — sample with test callers:

- `read-policy.yaml` — sample with test callers:
  ```yaml

    - checkout

  support-agent:
    - support
    - billing

    - billing
  ```

- `.env.example` — all env vars with placeholder values and comments.
  Must include `TILTH_IDENTITY=test-writer`.

Mounted into the gateway containers via docker-compose volumes.

### End-to-end test (`e2e/test_end_to_end.py`)

A pytest module that:

1. Assumes `make up` has been run (or starts the stack itself).
2. Tests the full pipe through the public APIs:

```python
def test_full_write_read_cycle():
    # Set client config for the test
    os.environ["TILTH_GATEWAY_URL"] = "http://localhost:8001"
    os.environ["TILTH_IDENTITY"] = "test-writer"

    from tilth import send

    send("Stripe returned card_declined for user 42",
         namespace="checkout", severity="warn")
    send("Customer requested refund cited shipping delay",
         namespace="support", severity="info")
    send("Invoice paid for subscription tier-2",
         namespace="billing", severity="info")

    # Wait for background worker + gateway batch to flush.
    time.sleep(3)

    # Query directly through the query gateway.
    resp = httpx.post(
        "http://localhost:8002/query",
        headers={"x-workload-identity": "ops-copilot"},
        json={"query": "payment failures", "top_k": 3},
    )
    assert resp.status_code == 200
    results = resp.json()["results"]
    assert len(results) > 0

    # Top result should be the checkout record.
    top = results[0]
    assert top["namespace"] == "checkout"
    assert "card_declined" in top["content"]
    assert "<retrieved_document" in top["content"]
    assert top["content_hash"] is not None

    # ACL enforcement: a caller restricted to billing shouldn't see
    # the checkout record, even when it's most relevant.
    resp = httpx.post(
        "http://localhost:8002/query",
        headers={"x-workload-identity": "billing-only-reader"},
        json={"query": "payment failures", "top_k": 3},
    )
    assert resp.status_code == 200
    for r in resp.json()["results"]:
        assert r["namespace"] == "billing"
```

## Behavior

### `make e2e`

1. `docker-compose up -d --build` — builds and starts qdrant + gateways.
2. Wait for `/healthz` on both gateways to return 200 (timeout 60s).
3. Run `pytest e2e/`.
4. `docker-compose down -v` regardless of test outcome.
5. Exit with the test exit code.

### Test isolation

Each e2e test should clean up after itself. Either:
- Use unique namespaces per test and filter on them, or
- Recreate the Qdrant collection in a fixture.

Tests must pass when run in any order, individually, or in a loop.

## Acceptance criteria

- [ ] `make install` succeeds on a clean clone with no manual steps.
- [ ] `make lint`, `make typecheck`, `make test` all pass.
- [ ] `make up` brings up Qdrant and both gateways, both report healthy
      within 60 seconds.
- [ ] `make e2e` runs to completion, tests pass, stack is torn down.
- [ ] `e2e/test_end_to_end.py::test_full_write_read_cycle` passes.
- [ ] `e2e/test_end_to_end.py::test_pii_scrubbing_end_to_end` passes.
- [ ] `e2e/test_end_to_end.py::test_mcp_search_tool_round_trip` passes.
- [ ] Running `make e2e` twice in a row works (no leftover state).
- [ ] `config/.env.example` documents every required env var with a comment.

## Out of scope

- CI configuration (GitHub Actions). Add separately.
- Performance benchmarks. Functional tests only for v1.
- Chaos testing. Post-v1.
- Multi-replica deployment manifests. Post-v1.
- TLS between services. Local dev uses plaintext.

## Notes

The e2e test is the most important test in the project. It proves the
components work as a system.

The `time.sleep(3)` for batch flushing is acceptable for v1. A better
pattern would be a `/flush` debug endpoint on the ingest gateway that
drains the batch queue synchronously, gated on a debug flag. If that's
easy, add it; if it's a yak shave, sleep is fine.

The `TILTH_IDENTITY=test-writer` env var is what makes e2e tests work
without a mesh. The client library sets `x-workload-identity: test-writer`
on every POST, and the write-policy.yaml authorizes `test-writer` for all
test namespaces. This is the resolution to the e2e auth gap — see
DECISIONS.md for the full rationale.
